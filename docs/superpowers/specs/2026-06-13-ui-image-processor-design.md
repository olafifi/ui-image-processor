# UI 图片处理器设计规格

日期：2026-06-13

## 目标

构建一个可部署到 GitHub Pages 的个人网页工具，用于游戏策划日常制作 UI 示意图时的轻量图片处理。工具优先满足“快速、稳定、够干净”的工作流，不追求专业修图软件级精度。

图片只在浏览器本地处理，不上传服务器。第一版面向桌面端 Chrome / Edge，自动抠图依赖 WebGPU；不满足条件时降级为基础图片处理与手动清理。

## 第一版范围

包含：

- 多图导入：按钮选择和拖拽导入。
- 导入格式：PNG、JPG/JPEG、WebP。
- SAM 自动抠图：桌面 Chrome / Edge 且 WebGPU 可用时启用。
- 手动清理：橡皮擦、恢复画笔。
- 裁剪：固定比例和自由比例。
- 圆角裁图。
- 背景：透明背景或纯色背景。
- 输出尺寸调整。
- 导出格式：PNG、JPG/JPEG。
- 单张导出。
- 批量 ZIP 导出。
- 本地模板：保存和套用稳定参数。
- CSV 往返批量重命名。
- 本地保存预设，不做账号、云同步或后端存储。

不包含：

- 拖入整个文件夹。
- 手机端完整适配。
- 服务器端 AI 抠图。
- 云端存储或跨设备同步。
- 把 SAM 点位、笔刷轨迹作为批量模板复用。

## 主流程

1. 用户打开网页。
2. 工具执行环境自检，判断是否启用 SAM 完整模式。
3. 用户点击导入或从文件夹拖入单张/多张图片。
4. 工具过滤非图片文件，并把 PNG、JPG/JPEG、WebP 统一解码为浏览器位图。
5. 左侧队列显示导入图片，用户选择当前编辑图片。
6. 用户在中央画布进行抠图、修补、裁剪、圆角和背景设置。
7. 用户可以保存模板或套用模板到当前图片/全部图片。
8. 用户可下载 CSV 命名模板，在表格软件中填入新文件名后再导入 CSV。
9. 用户导出当前 PNG/JPG，或下载批量 ZIP。

## UI 信息架构

采用 V6 布局：

- 大画布居中。
- 左侧窄队列。
- 右侧模板与导出面板。
- 顶部核心功能栏。

### 顶部核心功能栏

高频核心功能放在顶部，并强化视觉权重：

- 导入图片：主按钮，支持点击多选。
- 套用模板：打开模板选择。
- 批量重命名：打开 CSV 往返重命名流程。

顶部不放 `下载 ZIP`。`下载 ZIP` 属于收尾导出动作，放在右侧导出区。

### 中央画布

中央画布是第一视觉重心，占据工作区主要面积。图片应尽量铺满画布可编辑区域。

画布支持：

- 拖入单张或多张图片。
- 裁剪框显示。
- mask 预览。
- 透明棋盘格背景预览。
- 圆角预览。

### 工具栏分组

工具栏分为两组：

- 选择与修补：SAM 点选、反点选、橡皮擦、恢复画笔、裁剪。
- 历史：撤销、重做。

撤销和重做不与选择工具同级显示，使用圆形回退/前进箭头，接近 Photoshop 等成熟工具的表达。

图标规则：

- 不使用 `IN`、`TP`、`RN`、`ER`、`BR`、`CR`、`L`、`R` 等英文缩写占位。
- 图标必须使用成熟、直观、可识别的表达。
- 重要按钮使用“图标 + 中文标签”。
- 仅图标按钮必须有 tooltip。

### 底部裁剪与输出条

裁剪比例：

- 1:1
- 4:3
- 3:4
- 16:9
- 9:16
- 自由

其他输出控制：

- 圆角开关和圆角半径。
- 透明背景 / 纯色背景。
- 导出摘要，例如 `PNG · 1024 x 1024`。

### 左侧队列

左侧队列显示：

- 图片缩略图。
- 当前选中态。
- 图片数量。
- 拖拽导入提示。

第一版支持多张图片文件拖入，不支持文件夹拖入。如果用户拖入文件夹，提示打开文件夹后选择图片拖入。

### 右侧模板与导出面板

右侧面板显示：

- 当前模板。
- 输出尺寸。
- 背景设置。
- 命名状态。

收尾操作区包含：

- 保存模板。
- 导出当前 PNG/JPG。
- 下载 ZIP。

## 图像处理设计

### 导入与解码

支持导入：

- `.png`
- `.jpg`
- `.jpeg`
- `.webp`

导入后统一通过浏览器能力解码为 `ImageBitmap` 或等价 Canvas 可绘制对象。后续处理不直接依赖原始格式。

### SAM 抠图

第一版采用纯前端静态部署方案：

- 使用 ONNX Runtime Web / WebGPU 跑轻量 SAM 或 MobileSAM 类模型。
- 模型文件作为静态资源随站点部署。
- 图片不上传服务器。

启动自检：

- 检测 `navigator.gpu`。
- 检测必要浏览器能力，例如 Web Worker、Canvas。
- 尝试加载模型。
- 推理失败或超时时切换降级模式。

完整模式：

- 支持 SAM 点选。
- 支持反点选。
- 支持根据点位生成 mask。
- 支持将 mask 应用于透明背景。

降级模式：

- 自动抠图不可用。
- 保留裁剪、格式转换、背景、圆角、导出、批量重命名。
- 保留手动橡皮擦与恢复画笔。

### 手动修补

手动修补作用于 mask：

- 橡皮擦：从当前 mask 中删除区域。
- 恢复画笔：把区域加回 mask。
- 笔刷大小可调。
- 操作进入历史栈，支持撤销/重做。

### 裁剪与圆角

裁剪支持固定比例和自由比例。裁剪框用相对坐标保存，便于模板在不同图片尺寸上套用。

圆角在导出阶段应用：

- 透明背景时，圆角外部为透明。
- 纯色背景时，圆角外部填充所选背景色。

### 背景与导出格式

默认导出 PNG。

规则：

- PNG 支持透明背景和纯色背景。
- JPG/JPEG 不支持透明背景。
- 当背景为透明时，导出格式自动锁定 PNG。
- 当用户选择 JPG/JPEG 时，必须选择纯色背景，默认白色，可自定义颜色。

### 输出尺寸

导出时可设置输出宽高。固定比例裁剪时，根据裁剪比例约束宽高；自由裁剪时允许用户分别输入宽高。

第一版需要提供常用尺寸输入，不强制预设列表。

## 模板与本地记忆

模板保存在浏览器本地，优先使用 IndexedDB；必要时可用 LocalStorage 存储轻量配置。

模板保存稳定参数：

- 裁剪比例。
- 裁剪框相对位置。
- 输出尺寸。
- 圆角开关和半径。
- 背景类型和颜色。
- 导出格式。
- 命名规则。

模板不保存：

- SAM 点位。
- 橡皮擦轨迹。
- 恢复画笔轨迹。
- 单张图片专属 mask。

模板可套用到：

- 当前图片。
- 全部图片。

## CSV 往返批量重命名

批量重命名以 CSV 往返为第一版核心流程，不强迫用户在网页里逐个编辑。

流程：

1. 用户导入图片。
2. 用户点击 `批量重命名`。
3. 工具生成 `rename-template.csv`。
4. 用户在 Excel / WPS / 表格软件中填写新名字。
5. 用户把 CSV 拖回工具或点击上传 CSV。
6. 工具校验 CSV。
7. 导出当前图片或 ZIP 时按 CSV 中的新文件名输出。

CSV 字段：

```csv
index,old_filename,new_filename
1,hero_ref_01.webp,ui_button_start
2,icon_raw.png,ui_icon_coin
3,button_ref.jpg,ui_panel_reward
```

规则：

- CSV 使用 UTF-8 BOM，降低 Excel 中文乱码概率。
- `index` 用于辅助排序和校验。
- `old_filename` 必须能匹配导入队列中的文件。
- `new_filename` 可为空；为空时沿用旧文件名。
- `new_filename` 可以不写扩展名，工具按当前导出格式自动补扩展名。
- 如果写了扩展名，必须与当前导出格式一致。
- 非法文件名字符需要提示。
- 重名需要提示。
- CSV 行缺失、重复、无法匹配需要提示。

网页内可保留快速命名规则，例如 `ui_ref_{n}`，但它是辅助功能，不是主批量命名流程。

## 导出与 ZIP

单张导出：

- 导出当前图片。
- 使用当前图片的裁剪、mask、圆角、背景、格式、尺寸和命名设置。

批量 ZIP：

- 对队列中所有图片执行导出。
- 按 CSV 或命名规则确定文件名。
- 所有文件使用当前选定导出格式；透明背景时统一 PNG。
- 使用浏览器端 ZIP 生成库打包。

## 错误处理

需要处理并提示：

- 不支持的文件格式。
- WebP 解码失败。
- 图片过大导致处理失败或内存不足。
- WebGPU 不可用。
- SAM 模型加载失败。
- SAM 推理超时。
- CSV 格式错误。
- CSV 文件名无法匹配。
- CSV 新文件名重复。
- JPG 导出时仍选择透明背景。
- ZIP 生成失败。

错误提示原则：

- 明确说明发生了什么。
- 给出可执行下一步。
- 自动降级时说明当前处于降级模式，但不阻断基础功能。

## 数据模型

图片队列项：

- `id`
- `sourceFile`
- `originalName`
- `targetName`
- `mimeType`
- `bitmap`
- `naturalWidth`
- `naturalHeight`
- `mask`
- `crop`
- `exportSettings`
- `history`

裁剪参数：

- `ratio`: `1:1`、`4:3`、`3:4`、`16:9`、`9:16`、`free`
- `x`
- `y`
- `width`
- `height`
- 坐标使用相对值保存。

导出参数：

- `format`: `png` 或 `jpeg`
- `width`
- `height`
- `backgroundType`: `transparent` 或 `solid`
- `backgroundColor`
- `cornerRadius`

模板：

- `id`
- `name`
- `crop`
- `exportSettings`
- `namingRule`
- `createdAt`
- `updatedAt`

CSV 命名映射：

- `oldFilename`
- `newFilename`
- `status`
- `message`

## 技术架构

推荐技术栈：

- Vite
- React
- TypeScript
- Canvas 2D
- ONNX Runtime Web / WebGPU
- IndexedDB
- CSV 解析库
- ZIP 生成库
- GitHub Pages 静态部署

前端模块：

- `ImportManager`：处理按钮导入、拖拽导入、格式过滤和解码。
- `ImageQueue`：维护图片队列、选中态和缩略图。
- `EditorCanvas`：画布渲染、缩放、裁剪框和 mask 预览。
- `SegmentationEngine`：SAM 模型加载、能力检测和推理。
- `MaskEditor`：橡皮擦、恢复画笔和 mask 合成。
- `CropExporter`：裁剪、圆角、背景合成和 PNG/JPG 导出。
- `TemplateStore`：模板保存、读取、套用。
- `RenameCsvFlow`：CSV 生成、上传、解析、校验和映射。
- `ZipExporter`：批量导出和 ZIP 生成。

## 测试与验收

基础验收：

- 能在本地开发服务器打开页面。
- 能拖入多张 PNG/JPG/WebP。
- 能点击导入多张 PNG/JPG/WebP。
- WebP 能正常显示并导出为 PNG 或 JPG。
- 透明背景时导出格式锁定 PNG。
- JPG 导出时自动要求纯色背景。
- 裁剪比例包含 1:1、4:3、3:4、16:9、9:16、自由。
- 圆角导出效果正确。
- CSV 命名模板能下载。
- 填写后的 CSV 能上传并正确映射新文件名。
- 批量 ZIP 内文件名正确。
- 撤销/重做只影响编辑历史，不作为普通修补工具展示。
- SAM 不可用时能降级，基础功能仍可用。

视觉验收：

- 画布保持 V6 的大占比。
- 顶部核心功能视觉权重高。
- `下载 ZIP` 在右侧导出区，不在顶部。
- 所有主要功能有直观图标。
- 不出现英文缩写占位图标。

浏览器验收：

- 桌面 Chrome / Edge 完整模式。
- 不支持 WebGPU 的环境进入降级模式。

部署验收：

- 项目可构建为静态文件。
- 可通过 GitHub Pages 访问。
- 不需要后端服务。
